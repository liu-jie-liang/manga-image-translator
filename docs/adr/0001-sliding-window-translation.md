# ADR-0001: 滑动窗口翻译策略

## 状态

已提案 (Proposed)

## 背景

当前 `manga-image-translator` 的翻译流程是逐页独立翻译，仅通过 `--context-size` 参数提供前 N 页译文的简单拼接作为上文。这导致：

1. **上下文不足**：后续页面无法看到"后文"，翻译缺乏前瞻性
2. **首尾页质量差**：第1页没有上文，最后一页没有下文
3. **翻译一致性弱**：同一角色台词跨页时风格可能不一致

## 决策

引入**滑动窗口翻译策略**：以窗口为单位组包翻译，每个窗口 OCR 5 页文本并拼成一个 prompt，翻译结果取窗口中间页，向外逐步降权。窗口大小为 5 页（可配置）。

### 核心规则

| 规则 | 描述 |
|------|------|
| 窗口大小 | 5 页（`window_size = 5`） |
| 页分隔符 | `|||`（三个竖线） |
| 文本框编号 | 每页内文本框独立编号，跨页全局递增 `<\|N\|>` |
| 翻译赋值 | 每个窗口翻译完成后，取对应页的译文段落 |
| 渲染时机 | 首窗渲染前 `floor(window_size/2)` 页，中间窗各渲染 1 页，末窗渲染剩余页 |
| OCR 策略 | **预 OCR**：进入滑动窗口前，一次性 OCR 所有页面，缓存结果 |

### 窗口编排示例（10 页漫画）

```
窗口 [1-5]:
  OCR已缓存  → assemble("页1文本" ||| "页2文本" ||| "页3文本" ||| "页4文本" ||| "页5文本")
  Translate   → LLM 返回翻译
  Parse       → 提取页1,2,3的译文
  Render      → 渲染页1,2,3

窗口 [2-6]:
  assemble("页2文本" ||| "页3文本" ||| "页4文本" ||| "页5文本" ||| "页6文本")
  Translate → Parse → 提取页4译文
  Render    → 渲染页4

窗口 [3-7]:
  assemble(...页3-7...)
  Translate → Parse → 提取页5译文
  Render    → 渲染页5

窗口 [4-8]:
  assemble(...页4-8...)
  Translate → Parse → 提取页6译文
  Render    → 渲染页6

窗口 [5-9]:
  assemble(...页5-9...)
  Translate → Parse → 提取页7译文
  Render    → 渲染页7

窗口 [6-10]:
  assemble(...页6-10...)
  Translate → Parse → 提取页8,9,10译文
  Render    → 渲染页8,9,10
```

### Prompt 格式

```
<|1|>第1页文本框1的OCR文本
<|2|>第1页文本框2的OCR文本
|||
<|3|>第2页文本框1的OCR文本
<|4|>第2页文本框2的OCR文本
|||
<|5|>第3页文本框1的OCR文本
<|6|>第3页文本框2的OCR文本
|||
<|7|>第4页文本框1的OCR文本
<|8|>第4页文本框2的OCR文本
|||
<|9|>第5页文本框1的OCR文本
<|10|>第5页文本框2的OCR文本
```

### 翻译结果解析

LLM 返回结果同样为 `<|N|>` 标注，按 ID 号映射回对应页的文本框：

```
<|1|>中文译文1-1
<|2|>中文译文1-2
<|3|>中文译文2-1
...
```

### 配置参数

| 参数名 | 默认值 | 描述 |
|--------|--------|------|
| `--sliding-window` | `False` | 启用滑动窗口翻译模式 |
| `--sliding-window-size` | `5` | 滑动窗口大小（页数） |

## 影响

### 优点
- 每页翻译都能看到前后文，翻译质量和连贯性提升
- 首尾页翻译质量提高（末窗集中渲染最后几页）
- 代码模块化：滑动窗口作为独立模块，不影响现有翻译管道

### 缺点
- 总 API 调用次数 = `N - window_size + 1`（与逐页独立翻译次数相同）
- 每窗口 prompt 更长（单窗口内 5 页文本），约等于 5 个单独请求的 token 总和
- 需要预加载所有页的 OCR 结果，内存占用增加

### 稳定性考量
- 单窗口 prompt 过长时，LLM 可能截断或丢失部分译文 → 增加解析校验和重试
- 中间页被 3 个窗口重复翻译，取最后一次且上下文最完整的窗口结果

## 实现计划

1. 新建 `manga_translator/sliding_window.py` 模块
2. 在 `manga_translator.py` 的 `_run_text_translation` 中集成
