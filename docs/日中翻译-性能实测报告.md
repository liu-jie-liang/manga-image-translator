## 方式C 直接模型测试

| 用例 | 原文 | 译文 | 耗时 |
|------|------|------|------|
| 日常对话 | こんにちは、今日はいい天気ですね。 | 你好，今天天气真好呢。 | 1.1s |
| 批量翻译 | おはようございます / 今日も頑張りましょう / さようなら | 早安。 / 今天也加油吧。 / 再见。 | 0.9s |
| R18词汇 | 感じる / 気持ちいい / やめて | 有感觉了。 / 好舒服。 / 不要。 | 0.9s |

## 结论

1. **方式B 和方式C 速度几乎一致**：差异 <1%，Galtransl 模型未带来额外性能开销
2. **方式C 对 R18 内容翻译支持更好**：直接模型测试中 R18 词汇均正常翻译，无空白返回
3. **翻译阶段占总耗时约 60%**：是批处理的主要瓶颈，与模型选择无关
4. **方式C 与方式B 可互为补充**：方式B 适合一般漫画翻译，方式C 适合含 R18 内容的视觉小说/漫画
5. **推荐策略**：一般漫画使用方式B（降级链有保障），R18 内容使用方式C（专为 Galgame 优化）

### 结论

1. **方式B (Sakura GGUF) 最快**：全量 79.7s，比方式A 快 24%，与方式C 差异 <2%
2. **GGUF 直连显著优于 Ollama HTTP**：翻译阶段快 39%（3.53s vs 5.79s），零网络开销
3. **方式B 与方式C 速度几乎一致**：差异仅 1.2s（79.7s vs 80.9s），token 吞吐量接近（20.68 vs 20.72 tok/s）
4. **检测+OCR 阶段稳定**：三种方式检测 0.32-0.39s，OCR 1.17-1.72s，与翻译器选择无关
5. **续传机制全部正常**：三种方式均正确识别新图片、跳过已完成页面、progress 正确增长
6. **推荐策略**：优先方式B（最快+降级链保障），R18 内容使用方式C（专为 Galgame 优化）
