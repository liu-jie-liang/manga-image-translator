## 四、显存风险分析

### 4.1 显存分布（GPU 12GB）

```
GPU 12GB 总显存
├── 模型权重 (Q4KM):      ~11.0 GB   （固定不变）
├── KV Cache (2048 tok):   ~0.33 GB   （2160 tokens f16）
├── 推理激活缓冲区:        ~0.2-0.4 GB
└── 剩余安全余量:          ~0.3-0.5 GB
```

### 4.2 KV Cache 计算

- Sakura-14B = Qwen2.5-14B：40层，GQA 8 KV heads，head_dim=128
- 每 token KV Cache = 2 × 40 × 8 × 128 × 2B(f16) ≈ 164KB
- 满载（2048 tokens）= 2048 × 164KB ≈ 336MB
- 实测峰值 282 tokens → 仅 ~46MB

### 4.3 风险评估

| 场景 | 爆显存风险 | 说明 |
|------|-----------|------|
| 当前配置 (np=1, 2K) | **几乎为零** | KV cache + buffer < 剩余显存 |
| 扩大上下文到 4096 | **低** | KV cache 翻倍至~672MB，仍在安全范围 |
| 扩大上下文到 8192 | **中高** | KV cache ~1.3GB，可能超出剩余显存 |
| 调大 np > 1 | **高** | 多个并发请求各持一份 KV cache，显存暴增 |
